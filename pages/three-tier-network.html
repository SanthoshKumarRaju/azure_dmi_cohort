Three-Tier Network Architecture

Design secure network with load balancer, subnets, and network security groups

Architecture Overview

This project implements a secure three-tier network architecture on Azure with proper segmentation and load balancing:

Network Architecture

Web Tier
web-subnet
10.0.1.0/24
App Tier
app-subnet
10.0.2.0/25
DB Tier
db-subnet
10.0.3.0/26
Load Balancer
web-public-elb

Implementation Steps

1

Create Resource Group

Name: vnet-demo-rg
Region: Central India
Purpose: Three-tier network demo
2

Create Virtual Network

Set up the main virtual network with appropriate CIDR range:

Name: eb-demo-vnet
CIDR: 10.0.0.0/16
Subnets:
  - web-subnet: 10.0.1.0/24
  - app-subnet: 10.0.2.0/25
  - db-subnet: 10.0.3.0/26
  - AzureBastionSubnet: 10.0.4.0/26
3

Create Web Tier VM

Deploy Ubuntu VM in the web subnet:

VM Name: web-nginx
Image: Ubuntu 22.04 LTS
Subnet: web-subnet
Public IP: Enabled
Inbound Ports: SSH (22), HTTP (80)
4

Install & Configure NGINX

Connect to the VM and install NGINX web server:

# Connect to VM
ssh username@<public-ip>

# Install NGINX
sudo apt update
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx

Verify installation by visiting: http://<VM_PUBLIC_IP>/

5

Create Public Load Balancer

Configure a public load balancer for high availability:

Name: web-public-elb
Type: Public
SKU: Standard
Frontend IP: web-elb-ip (new public IP)
Backend Pool: web-backend-pool
Health Probe: TCP port 80
Load Balancing Rule: TCP 80 → 80
6

Configure Network Security Groups

Set up NSGs for each subnet with appropriate rules:

Web Subnet NSG

# Inbound Rules
Allow HTTP (80) - Source: Internet
Allow SSH (22) - Source: YourIP
Deny all other inbound

App Subnet NSG

# Inbound Rules
Allow All - Source: web-subnet
Deny internet direct access

DB Subnet NSG

# Inbound Rules
Allow MySQL (3306) - Source: app-subnet
Deny all other access
7

Test the Architecture

Verify the complete setup by accessing the application through the load balancer:

# Test URL
http://4.240.104.251/

You should see the NGINX default page served through the load balancer.

Security Considerations

Network Segmentation

  • Each tier in separate subnet with specific CIDR ranges
  • Proper route tables for inter-tier communication
  • Network Security Groups for traffic control

Access Control

  • Web tier accessible from internet on port 80/443 only
  • App tier accessible only from web tier
  • DB tier accessible only from app tier
  • SSH restricted to specific IP ranges
