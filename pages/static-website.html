Static Website on Azure Storage

Host cost-effective static websites using Azure Blob Storage with custom domain

Project Overview

Azure Storage static website hosting provides a serverless, cost-effective solution for hosting static content like HTML, CSS, JavaScript, and image files.

Static Website Architecture

HTML/CSS/JS
Static Files

CDN

Implementation Steps

1

Create Resource Group

Name: static-web-hosting
Region: Your preferred region
Purpose: Static website hosting resources
2

Create Storage Account

Create a new storage account with specific configuration:

# Storage Account Configuration
Name: staticwebhosting1 (globally unique)
Performance: Standard
Redundancy: LRS (Locally-redundant storage)
Hierarchical namespace: Disabled
3

Enable Static Website

Enable static website hosting in the storage account:

# Navigation in Azure Portal
Storage Account → Data management → Static website

# Configuration
Static website: Enabled
Index document name: index.html
Error document path: 404.html

This automatically creates a $web container for your files.

4

Prepare Website Files

Clone and customize the website repository:

# Clone repository (if using existing project)
git clone <your-repository-url>
cd <project-folder>

# Customize content
# Update text, images, and styling as needed

Ensure you have at least an index.html file.

5

Upload Files to Storage

Upload all website files to the $web container:

# Using Azure Portal
Storage Account → Containers → $web → Upload

# Select all files (HTML, CSS, JS, images)
# Maintain folder structure if any

Alternatively, use Azure CLI or Azure Storage Explorer for bulk uploads.

6

Access Your Website

Your static website is now live! Access it using the provided URL:

# Primary endpoint URL format
https://<storage-account-name>.z29.web.core.windows.net/

# Actual deployed URL
https://staticwebhosting1.z29.web.core.windows.net/
Website Live: Your static website is now accessible globally!

Cost Analysis

Pricing Benefits

  • No VM costs: Serverless architecture eliminates compute expenses
  • Pay-per-use storage: Only pay for actual storage used
  • Low egress costs: Data transfer costs are minimal
  • Free tier: First 5 GB storage and certain operations free

Estimated Monthly Cost (Small Website)

  • Storage: 1 GB ≈ $0.02
  • Operations: 10,000 reads ≈ $0.04
  • Data transfer: 10 GB ≈ $0.87
  • Total: ≈ $0.93/month
