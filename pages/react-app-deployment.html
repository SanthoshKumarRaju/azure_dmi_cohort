React App on Azure VM

Deploy a React application on Ubuntu VM with Nginx web server configuration

Project Overview

This project demonstrates how to deploy a production-ready React application on an Azure Virtual Machine running Ubuntu Linux, using Nginx as a web server.

Implementation Steps

1

Create Resource Group

Create a new resource group to organize all related resources:

# Resource Group Details
Name: react-app-rg
Region: West Europe
Purpose: React application deployment
2

Provision Ubuntu VM

Create a Ubuntu Virtual Machine with the following configuration:

# VM Configuration
Image: Ubuntu 20.04 LTS
Size: B1s (1 vCPU, 1 GB RAM)
Username: azureuser
Authentication: SSH Public Key
Inbound Ports: SSH (22), HTTP (80)
3

Connect to VM via SSH

Connect to your VM using SSH:

# SSH Connection Command
ssh azureuser@<VM_PUBLIC_IP>

Replace <VM_PUBLIC_IP> with your VM's actual public IP address.

4

Install Node.js & npm

Update system packages and install Node.js runtime:

# Update system and install Node.js
sudo apt update && sudo apt upgrade -y
sudo apt install -y nodejs npm

# Verify installation
node -v
npm -v
5

Clone & Build React App

Clone the React application repository and build it:

# Clone repository
cd ~
git clone https://github.com/pravinmishraaws/my-react-app.git
cd my-react-app

# Install dependencies and build
npm install
npm run build
6

Install & Configure Nginx

Install Nginx and configure it to serve the React build:

# Install Nginx
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx

# Configure Nginx
sudo nano /etc/nginx/sites-available/default

Update the Nginx configuration:

# Nginx configuration for React app
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    server_name _;

    location / {
        root /home/azureuser/my-react-app/build;
        index index.html index.htm;
        try_files $uri /index.html;
    }
}
7

Test & Restart Services

Test the Nginx configuration and restart the service:

# Test configuration and restart
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl status nginx --no-pager
