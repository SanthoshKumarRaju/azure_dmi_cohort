2. Deploy a React Application on an Azure Virtual Machine

Intermediate

Deployed a React application on an Ubuntu VM using Nginx as a web server. This project demonstrates infrastructure provisioning, application deployment, and web server configuration on Azure.

Implementation Steps:

  1. Created Resource Group: react-app-rg
  2. Provisioned Ubuntu 20.04 LTS VM (B1s size)
  3. Configured inbound ports for SSH (22) and HTTP (80)
  4. Installed Node.js and npm
  5. Cloned and built React application from GitHub
  6. Configured Nginx for static hosting
  7. Tested the deployment

3. Design a Three-Tier Network with Load Balancer on Azure

Advanced

Implemented a secure three-tier network architecture with web, application, and database tiers separated by subnets and protected by network security groups. Added a public load balancer for high availability.

Architecture Components:

  • Resource Group: vnet-demo-rg
  • Virtual Network: eb-demo-vnet (10.0.0.0/16)
  • Subnets:
    • web-subnet (10.0.1.0/24)
    • app-subnet (10.0.2.0/25)
    • db-subnet (10.0.3.0/26)
  • Web Tier VM: web-nginx (Ubuntu 22.04)
  • Load Balancer: web-public-elb (Standard SKU)
  • Health Probe & Load Balancing Rules

4. Host a Static Website on Azure Storage

Intermediate

Deployed a cost-effective static website using Azure Blob Storage static website hosting feature. This approach eliminates the need for web servers and provides high availability at a lower cost.

Implementation Steps:

  1. Created Resource Group: static-web-hosting
  2. Cloned and customized the website repository
  3. Created Azure Storage account with static website hosting enabled
  4. Uploaded all website files to the storage container
  5. Accessed the website via the static website URL

5. Deploy EpicBook on Azure VM + Azure Database for MySQL

Advanced

Deployed a full-stack application (EpicBook) with a web frontend on an Azure VM and database backend on Azure Database for MySQL Flexible Server with private network integration.

Architecture Details:

  • Resource Group: EpicBook-RG
  • Virtual Network: epic-vnet (10.0.0.0/16)
  • Subnets:
    • public-subnet (10.0.1.0/24) - for VM
    • mysql-subnet (10.0.2.0/24) - for MySQL
  • Web VM: epicbook-vm (Ubuntu 22.04)
  • Database: Azure MySQL Flexible Server (Private access)
  • Network Security Groups for traffic control
  • Nginx as reverse proxy
